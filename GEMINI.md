# Проект: Личный сайт-визитка

## 1. Общее описание

Этот проект представляет собой простой статический личный сайт.

**Цель:** Создание сайта-визитки для предоставления в банк при открытии бизнес-счета для ИП.
**Содержание:**
-   Резюме / информация об авторе.
-   Контактная информация и призыв к действию (например, "напишите мне на email", "купите мои услуги").

## 2. Технологический стек

-   **HTML:** для структуры сайта.
-   **CSS:** для стилизации.
-   **JavaScript (опционально):** для добавления интерактивности, если потребуется.

Проект должен быть максимально простым и статическим, чтобы избежать сложностей с бэкендом и базами данных.

## 3. Хостинг и домен

-   **Хостинг:** Для статических сайтов отлично подходят следующие бесплатные или недорогие варианты:
    -   **GitHub Pages:** Бесплатный хостинг прямо из вашего репозитория на GitHub.
    -   **Netlify:** Предлагает бесплатный тариф для персональных проектов с автоматической сборкой и развертыванием из Git.
    -   **Vercel:** Аналогичен Netlify, также с щедрым бесплатным тарифом.
-   **Домен:** Доменное имя можно зарегистрировать у любого аккредитованного регистратора (например, GoDaddy, Namecheap, Google Domains). После покупки его можно будет легко подключить к выбранному хостингу.

## 4. Рекомендации по созданию фронтенда

Чтобы сайт отлично выглядел на любых устройствах (десктопы, планшеты, смартфоны), следуйте этим рекомендациям:

### а. Mobile-First подход

Начинайте разработку стилей с мобильной версии сайта. Это заставит вас сфокусироваться на самом важном контенте и функциональности. Затем, с помощью медиа-запросов, добавляйте стили для больших экранов.

### б. Адаптивная верстка (Responsive Design)

Используйте CSS медиа-запросы (`@media`), чтобы применять разные стили для разных размеров экрана.

**Пример:**
```css
/* Стили для мобильных устройств (по умолчанию) */
.container {
  width: 100%;
  padding: 15px;
}

/* Стили для планшетов */
@media (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
  }
}

/* Стили для десктопов */
@media (min-width: 1200px) {
  .container {
    width: 1170px;
  }
}
```

### в. Используйте гибкие макеты (Fluid Layouts)

Вместо фиксированных размеров в пикселях (`px`) используйте относительные единицы, такие как проценты (`%`), `vw` (viewport width), `vh` (viewport height), `em` и `rem`. Это поможет элементам на странице гибко подстраиваться под размер экрана.

### г. Оптимизация изображений

-   **Сжимайте изображения:** Используйте онлайн-сервисы (например, TinyPNG) или инструменты сборки для уменьшения веса изображений без значительной потери качества.
-   **Адаптивные изображения:** Используйте тег `<picture>` или атрибут `srcset` для тега `<img>`, чтобы браузер мог загружать изображение подходящего размера в зависимости от разрешения экрана.

### д. CSS Фреймворки (Опционально, но рекомендуется для скорости)

Чтобы не писать все с нуля, можно использовать готовый CSS фреймворк. Они уже содержат сетку, адаптивные компоненты и утилиты.
-   **Bootstrap:** Один из самых популярных фреймворков. Прост в освоении, имеет огромное количество готовых компонентов.
-   **Tailwind CSS:** Utility-first фреймворк, который дает больше гибкости, но требует некоторого привыкания.

### е. Кросс-браузерная совместимость

Тестируйте сайт в нескольких популярных браузерах (Chrome, Firefox, Safari), чтобы убедиться, что он везде выглядит корректно. Используйте `autoprefixer` (часто встроен в инструменты сборки), чтобы автоматически добавлять вендорные префиксы к CSS-свойствам.
